Use cases

こんなときに

日常のなかの「見られたくない」瞬間を、設定なしで守る。

電車・カフェ

隣に人が来た瞬間に

視線を感じてホーム画面に戻すだけで、Umbric は自動で画面をカモフラージュ。タスク切替の一覧にも実画面は残りません。

端末を渡すとき

"ちょっと見せて"に

誰かに端末を渡す瞬間でも、内緒モードのタブ・履歴・ブックマークは Face ID の中。通常モードだけが見える状態に保てます。

検索を残したくない

調べものの後に

赤シールドをワンタップで、Cookie・履歴・キャッシュ・最近閉じたタブを一括消去。セッション間にも自動で残りません。

Two axes

2 つの軸で守る

"残さない" と "見せない" は別の問題。Umbric はその両方を初期状態で守ります。

01 — DATA

痕跡を残さない

Cookie・キャッシュ・Web ストレージはセッション間に残らず、外部送信もゼロ。閲覧履歴は端末内のみで、いつでもワンタップで一括消去できます。

02 — PHYSICAL

見せない

覗き見・画面を渡す一瞬・偽装を想定した、能動的に「見せない」設計。バックグラウンドに移した瞬間、画面は自動でカモフラージュされます。

Features

主な機能

3 つの中心機能 + 細部まで作り込まれた防御層。

自動カモフラージュ

バックグラウンド遷移と同時に、現在のページは別のダミー表示に置き換わります。覗き見・タスク切替・スクリーンショットへの一次防御。

ワンタップ一括消去

アドレスバー横の赤シールドで、タブ・最近閉じたタブ・Cookie・キャッシュ・履歴を範囲指定で一気に削除できます。

Face ID 内緒モード

通常モードと完全に分離された、Face ID 保護下の独立した閲覧領域。タブ・履歴・ブックマークもそれぞれ別管理。

その他の機能

  • Cookie・キャッシュを非永続化（セッション間に残らない）
  • 閲覧履歴は端末内のみ、外部送信なし
  • 広告とトラッカーを 2 段階で遮断（ネットワーク + DOM）
  • Cookie 同意バナーを自動 reject（主要 14 種 CMP 対応）
  • URL から追跡パラメータを自動除去（utm_, fbclid, gclid 等）
  • 指紋採取対策（Canvas / WebGL / AudioContext / Navigator）
  • HTTPS 強制 / 詐欺サイト警告 / JS ダイアログ遮断
  • Lockdown Mode 強制（Umbric だけに iOS 相当の最強制限を適用）
  • EasyList Japan 含む 7 本のフィルタリストを同梱
  • Focus フィルター連携で内緒モードを自動切替
  • リーダーモード（読み上げ・要約対応）/ 最近閉じたタブ復元
  • 12 言語対応（アプリ単位での言語切替）

Why

なぜ作ったか

プライベートな検索を、誰にも見られず、誰にも保存されないまま終わらせたかった。

既存のブラウザにも「プライベートタブ」や「履歴削除」はある。けれどそれらは "毎回意識する" 操作で、忘れた瞬間に痕跡は残る。さらに、設定では守れない領域がある——隣に座った人の視線、家族に端末を渡す一瞬、App Switcher に映る画面。

Umbric は "残さない・見せない" を初期状態にした iOS ブラウザです。Apple 純正のプライバシー保護機能を最大限活用しつつ、覗き見・端末共有・偽装といった "物理的に見られる" 状況にも能動的に対応します。

— MRCY

FAQ

よくある質問

Safari のプライベートタブと何が違いますか？
Safari Private は履歴を残さないだけです。Umbric はそれに加えて、バックグラウンド時の自動カモフラージュ、Face ID で守られた内緒モード、ネットワーク層と DOM 層の 2 段階ブロック、Canvas / WebGL / AudioContext レベルの指紋採取対策、ワンタップ範囲指定消去など、能動的な防御層を備えています。
本当に追跡されない？
Cookie・キャッシュ・Web ストレージはセッション間に残しません。閲覧履歴は端末内のみで、外部への送信はゼロです。広告ネットワークによるフィンガープリンティングも、Canvas / WebGL / AudioContext / Navigator など複数のレイヤーで抑制しています。
バックグラウンドカモフラージュって何ですか？
アプリをホーム画面に戻したり、App Switcher を開いた瞬間、現在のページが別のダミー画面に自動で置き換わる機能です。覗き見・ショルダーハッキング・他人に端末を渡したときの一次防御として機能します。
内緒モードと通常モードは何が違いますか？
内緒モードは Face ID で保護された、完全に独立した閲覧領域です。タブ・履歴・ブックマークも通常モードとは別管理。Focus フィルターと連携させれば、特定の集中モードに入ったときだけ自動で切り替えることもできます。
Free のままでも十分守られますか？
プライバシーの土台はすべて Free に含まれます。Cookie 非永続化・バックグラウンドカモフラ・内緒モード・広告と追跡の 2 段階遮断・指紋採取対策・Cookie 同意自動 reject など、コアの防御は買わなくても全員に効きます。Pro は「自分の手で守りを引き直す」コントロール体験を重ねる層です。
Cookie 同意バナーを自動 reject にして支障はありませんか？
主要 14 種の Cookie 管理プラットフォーム (CMP) を対象に、最小限の同意で閉じます。サイトの一部機能が動かなくなるケースもまれにあるため、サイト別 allowlist で「このサイトだけ自動 reject を外す」設定が可能です。
アップデートは無料ですか？
アプリのバージョンアップによる新機能・互換性アップデートはすべて無料です。Pro も買い切り型のため、購入後の追加課金やサブスクリプションは発生しません。

Name

名前について

"Umbric"（アンブリック）は、ラテン語の Umbra（影）と Trick（さりげない手立て）から生まれた造語。画面の隣にあっても出しゃばらず、見られそうな瞬間にはさっと姿を隠して、自分のプライバシーを自分の手で守る——そのコンセプトをそのまま体験にしたブラウザです。

痕跡を残さないのは、誰かを欺くためではなく、自分のプライベートを自分で守るためです。
